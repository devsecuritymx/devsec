Documentación

devsec.sh

Framework de auditoría de firmware para microcontroladores ESP32 y ESP8266. Adquisición, análisis estático, scoring de riesgo y visualización web desde un único CLI.

./devsec.sh --help
Comandos

Todos los subcomandos disponibles con sus opciones, descripciones y ejemplos de uso.

scan
core output

Comando principal. Conecta al dispositivo por puerto serie, descarga la flash completa, extrae la partición de aplicación, calcula hashes SHA256, extrae strings y calcula un score de riesgo basado en hallazgos. Acepta la mayoría de flags del framework.

./devsec.sh scan [--port DEV] [--full|--fast|--forensic] [flags...]
# ./devsec.sh scan --port /dev/ttyUSB0 --full --tag produccion
# ./devsec.sh scan --fast --no-ghidra --format json -v
# ./devsec.sh scan --forensic --dump-size 0x200000 --output /tmp/audit
# ./devsec.sh scan --dry-run --full --port /dev/ttyACM0
analyze
output

Muestra el reporte del scan más reciente encontrado en devsec-out/. Por defecto imprime el reporte en texto plano. Con --json devuelve el JSON estructurado, útil para scripting o integración con otras herramientas.

./devsec.sh analyze [--json]
# ./devsec.sh analyze
# ./devsec.sh analyze --json | jq '.risk'
report
output

Imprime el reporte en texto plano del último scan. Equivalente a analyze sin --json. Respeta el flag --format para elegir entre texto o JSON. Ideal para exportar o guardar como evidencia.

./devsec.sh report [--format txt|json]
# ./devsec.sh report
# ./devsec.sh report --format json > resultado.json
list
util

Lista todos los scans almacenados en devsec-out/ con su chip, score de riesgo, nivel de riesgo y tag si fue asignado. Los colores indican el nivel de riesgo: verde=BAJO, amarillo=MEDIO/ALTO, rojo=CRITICO.

./devsec.sh list
# ./devsec.sh list
clean
danger

Elimina scans con más de N días de antigüedad. Por defecto elimina los de más de 30 días. Operación destructiva e irreversible. Usa --dry-run previamente para ver qué se eliminaría.

./devsec.sh clean [--days N]
# ./devsec.sh clean --days 7
# ./devsec.sh clean --days 30
web
web

Lanza el servidor web de DevSecWeb (FastAPI + uvicorn) en 127.0.0.1. Sirve el dashboard interactivo con visualización de scans, findings, gráfica de riesgo, tabla de particiones y buscador de strings. Con --open abre el navegador automáticamente.

./devsec.sh web [--port-web PORT] [--open]
# ./devsec.sh web
# ./devsec.sh web --port-web 9090 --open
met
util

Imprime el pipeline de metodología en terminal: las 7 fases del proceso de auditoría, desde la adquisición del firmware hasta la visualización en el dashboard. Útil para presentaciones o como recordatorio del flujo de trabajo.

./devsec.sh met
# ./devsec.sh met
Todos los flags

Referencia completa de parámetros. La mayoría aplica al comando scan y se propagan internamente a esp.sh.

Conexión
--port, -p <DEV>autoPuerto serie del dispositivo. Ej: /dev/ttyUSB0, /dev/ttyACM0
--baud, -b <RATE>autoFuerza un baud rate específico. Si no se indica, el wrapper prueba 921600 → 115200 automáticamente.
--chip <MODELO>autoFuerza el modelo de chip. Valores: ESP32 | ESP8266. Omite la detección automática.
--timeout, -t <SEG>120Tiempo límite en segundos para el proceso de dump. Si se supera, el proceso falla con error.
Modo de operación
--full—Dump completo de 4MB + lanza Ghidra headless al finalizar para análisis de reverse engineering.
--fast—Dump reducido de 1MB. Omite forensics del sistema. Ideal para análisis rápidos.
--forensic—Solo recolecta datos forenses del sistema (dmesg, lsusb, uname). No hace dump de flash.
--report-onlyfalseNo ejecuta dump. Regenera el reporte con los datos existentes en el workspace actual.
--dry-runfalseMuestra el comando completo que se ejecutaría (incluyendo todos los flags resueltos) sin ejecutar nada.
Dump
--dump-size <HEX>0x400000Tamaño del dump en hexadecimal. Ej: 0x200000 (2MB), 0x100000 (1MB). Sobreescribe el default del modo.
Análisis
--strings-min <N>6Longitud mínima de strings a extraer con el comando strings. Valores bajos generan más resultados.
--no-stringsfalseOmite la extracción de strings. Implica que el análisis de riesgo no tendrá datos de texto.
--no-analyzefalseOmite el cálculo del score de riesgo. Se generará un reporte con score 0.
--no-forensicsfalseOmite la recolección forense del sistema (dmesg, lsusb, uname). Acelera el proceso.
--no-hashfalseOmite el cálculo de hashes SHA256 del dump y del app bin.
--no-ghidrafalsePreviene el lanzamiento de Ghidra headless incluso en modo --full.
Output
--output, -o <DIR>devsec-outDirectorio raíz donde se almacenará el workspace del scan. Se creará un subdirectorio con timestamp.
--jsonfalseEn analyze y report: imprime el JSON estructurado en lugar del texto plano.
--format <FMT>txtFormato del reporte generado. Valores: txt | json | both. Con both se crean ambos archivos.
--tag <LABEL>—Etiqueta identificadora incluida en el reporte y visible en list. Ej: produccion, prototipo-v2
--note <TEXTO>—Nota libre incluida en el reporte. Útil para contexto del auditor o hallazgos manuales previos.
Web
--port-web <PORT>8000Puerto TCP donde se levantará el servidor web del dashboard.
--openfalseAbre el navegador predeterminado automáticamente apuntando al dashboard al iniciar el servidor.
Limpieza
--days <N>30Antigüedad en días para el comando clean. Elimina scans más antiguos que N días.
UX / Control
--verbose, -vfalseActiva salida detallada con líneas de debug (DBG) mostrando valores internos y decisiones del script.
--quiet, -qfalseModo silencioso. Suprime toda salida excepto errores fatales. Ideal para cron o pipelines CI.
--version—Imprime la versión del CLI y sale inmediatamente.
--help, -h—Muestra la ayuda en terminal. Con --help --web abre esta documentación en el navegador.
Pipeline

Las 7 fases del proceso de auditoría ejecutadas secuencialmente por esp.sh.

1
Acquisition
Dump completo de la flash del microcontrolador via esptool.py. El wrapper devsec_esptool.py intenta automáticamente múltiples baud rates (921600 → 115200) hasta completar la lectura.
2
Processing
Extracción de la partición de aplicación (offset 0x10000). Validación de integridad del dump (detección de dumps >90% 0xFF). Cálculo de hashes SHA256 del firmware completo y de la app.
3
Forensics
Recolección de información del host: dmesg, lsusb, uname -a. Se intenta con privilegios estándar y, si es necesario, con sudo. Se omite en modo --fast.
4
Analysis
Extracción de strings con longitud mínima configurable. Análisis heurístico: certificados, keys/tokens, endpoints HTTP/S, contraseñas, credenciales WiFi/SSID, brokers MQTT/AMQP, claves privadas. Score ponderado 0–80+.
5
Reverse Engineering
Lanzamiento de Ghidra headless en background para análisis de binario. Solo se ejecuta en modo --full y si ghidraRun está disponible en el PATH. Puede desactivarse con --no-ghidra.
6
Reporting
Generación de reporte estructurado en texto plano y/o JSON. Incluye chip, score, nivel de riesgo, modo, tag, nota, puerto y timestamp ISO. Configurable con --format txt|json|both.
7
Visualization
Dashboard web interactivo (FastAPI + Chart.js). Visualiza scans, findings, gráfica doughnut del score por nivel de riesgo, tabla de particiones ESP32 y buscador de strings en tiempo real.
Quick Reference

Comandos frecuentes listos para copiar.

./devsec.sh scan --full --tag prod
Audit completo con Ghidra
./devsec.sh scan --fast --no-ghidra
Scan rápido sin RE
./devsec.sh scan --dry-run --full
Ver flags sin ejecutar
./devsec.sh analyze --json | jq .
Reporte en JSON parseado
./devsec.sh web --port-web 9090 --open
Dashboard en puerto custom
./devsec.sh list
Ver todos los scans
./devsec.sh clean --days 7
Limpiar scans de >7 días
./devsec.sh scan --format both -q
Scan silencioso, output txt+json
./devsec.sh scan --strings-min 4 -v
Strings cortas + debug
./devsec.sh scan --no-forensics --no-hash
Solo dump y análisis
./devsec.sh scan --chip ESP32 --baud 115200
Chip y baud forzados
./devsec.sh report --format json > out.json
Exportar reporte a archivo
